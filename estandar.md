

| Entregable / Criterio | Insuficiente (1) | Regular (2) | Bueno (3) | Excelente (4) | Ponderación |
| :---: | :---: | :---: | :---: | :---: | :---: |
| E1: Repo \+ pipeline \+ preview | No hay repositorio operativo ni pipeline funcional. | Repositorio creado pero sin CI/CD o preview funcional. | Repo y pipeline funcionales, preview parcial o con errores. | Repo operativo, pipeline CI/CD y preview por PR funcionando correctamente. | 10% |
| E2: Landing \+ vistas clave responsivas | Landing incompleta, sin maquetado semántico ni responsive. | Landing básica, maquetado poco semántico o sin accesibilidad. | Landing semántica, responsive y accesible, pero con detalles a mejorar. | Maquetado semántico, responsive, accesible y visualmente consistente. | 15% |
| E3: Formularios dinámicos con fetch \+ validación | No hay formularios dinámicos ni validación ni fetch. | Formularios básicos, validación mínima o fetch incompleto. | Formularios dinámicos, validación y fetch integrados, pero con fallos menores. | DOM dinámico, validación robusta y fetch integrado correctamente. | 15% |
| E4: Catálogo navegable \+ API básica | No hay catálogo ni API interna funcional. | Catálogo básico, API incompleta o sin lógica funcional. | Catálogo navegable y API funcional, pero con lógica mejorable. | Catálogo \+ API interna, lógica funcional y checklist en PRs. | 20% |
| E5: CRUD funcional en Supabase \+ admin | BD no modelada, sin CRUD ni persistencia. | BD modelada, CRUD parcial o sin persistencia real. | CRUD funcional, persistencia y panel admin básico. | BD modelada, CRUD completo, persistencia y admin funcional. | 20% |
| E6: Checkout \+ webhook funcionales | No hay integración de pagos ni webhooks ni demo. | Pagos o webhooks integrados parcialmente, demo incompleta. | Pagos y webhooks funcionales, demo operativa con detalles a mejorar. | Pagos \+ webhooks, demo operativa y pruebas exhaustivas. | 20% |
| Funcionalidad (transversal) | La app no cumple los objetivos mínimos ni es funcional. | Funcionalidad parcial, errores frecuentes o flujos incompletos. | Funcionalidad mayormente completa, con detalles menores. | Funcionalidad completa, robusta y alineada a los objetivos. | 40%\* |
| Código / Estructura | Código desorganizado, sin estructura ni buenas prácticas. | Estructura básica, pero con problemas de organización o legibilidad. | Código organizado, legible y con buenas prácticas, pero mejorable. | Código limpio, modular, estructurado y siguiendo buenas prácticas. | 20%\* |
| Interfaz / Accesibilidad | Interfaz poco usable, sin accesibilidad ni coherencia visual. | Interfaz básica, accesibilidad mínima o visual poco consistente. | Interfaz usable, accesible y visualmente coherente, con detalles a mejorar. | Interfaz intuitiva, accesible y visualmente atractiva. | 15%\* |
| Despliegue | No hay despliegue ni acceso público al proyecto. | Despliegue parcial, acceso limitado o con errores. | Despliegue funcional, pero con detalles a mejorar. | Despliegue público, estable y accesible (Vercel, GitHub Pages, etc.). | 15%\* |
| Documentación | No hay documentación o es insuficiente. | Documentación básica, incompleta o desactualizada. | Documentación clara y suficiente, pero mejorable. | Documentación completa, clara y alineada al proyecto. | 10%\* |
