# Explicaciones técnicas

Esta carpeta guarda las explicaciones técnicas de cada decisión de código del proyecto. Cumple dos funciones:

1. **Material de repaso para el oral** — concentra los conceptos que pueden preguntarse en el parcial (HTML semántico, accesibilidad, responsive, eventos, async/await, fetch, validación, módulos ES6, componentes, props, state, useEffect, rutas, cliente vs servidor, CI/CD).
2. **Apoyo a la documentación de prompts** — los prompts en `docs/prompts/` cuentan **qué se hizo**; estas explicaciones cuentan **por qué funciona** y los conceptos detrás.

## Convención de nombres

`NN-tema.md` (numeración correlativa por orden de aparición del concepto).

Ejemplos previstos:
- `01-html-semantico.md`
- `02-css-responsive.md`
- `03-eventos-y-dom.md`
- `04-asincronia-fetch.md`
- `05-modulos-es6.md`
- `06-componentes-react.md`
- `07-props-y-state.md`
- `08-useeffect.md`
- `09-rutas-next.md`
- `10-cli-cd-vercel.md`

Cada archivo debe poder leerse de forma autónoma, en lenguaje claro, con un ejemplo del propio proyecto.
